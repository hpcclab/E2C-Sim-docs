Task Report

This option is if you want to view a more task centric report; everything in the task report is included in the full report. It includes:

  • id: Each task has an id number unique to itself for identification purposes.
  • Task type: Tasks can be of a certain type, denoted as T1, T2, T3, etc. “Task types” for the purpose of this simulator are an abstract concept that correlate to different real-world applications.
  • Status: The status of each task describes their outcome in the simulation, such as completed or missed (missed deadline).
  • Assigned machine: Which machine the scheduler submitted a particular task to.
  • Arrival time: When the task arrived at its submitted machine.
  • Start time: When a machine began its processing of a task.
  • Completion time: When a machine finished its processing of a task. Only available if the task was not canceled or missed.
  • Missed time: Only available if the task missed its deadline. Displays the time the task took to reach its deadline.
  • Execution time: Shows the time a task would take on any given machine (a task is assigned to only one machine by the scheduler during the simulation, but has an expected execution time written for all machines).
