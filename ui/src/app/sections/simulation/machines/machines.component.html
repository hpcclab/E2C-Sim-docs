Machines

Machines in this simulator are analogous to the individual machines that make up a computer cluster. They can be heterogeneous or homogeneous machines. If machines are homogeneous, during the simulation run time they will appear as the same color.

If the system is heterogeneous, the machines will appear as different colors during the run time.

More details about heterogeneity and homogeneity are given below in the Machine Configuration section.

If you click on the circle icon that represents a machine, you may view its:

  • ID (identification number)
  • Power
  • Queue Size (how many tasks total can fit in the machine queue - more details about Queue Sizes are in the Machine Queues section below)

You can also view each machines performance during the simulation run time, which includes:

  • # of Assigned Tasks (how many tasks a machine has been assigned so far throughout the simulation)
  • # of Completed Tasks (how many tasks a machine has successfully completed throughout the simulation)
  • # of Missed Tasks (how many tasks a machine has missed the deadline to complete throughout the simulation)
  • Energy Usage (consumed energy by the machine)
  • Wasted Energy (energy wasted working on tasks that missed their deadline)
  • Idle Energy Usage
During the simulation run time, if a task is currently allotted to a machine, you can click the square icon representing the task to get more information about it.

Machine Configuration

  1. Heterogeneity vs Homogeneity

    A group of machines can be a heterogeneous or homogeneous system.

    A heterogeneous system is one composed of different machines. In the real world, this would mean each machine having different hardware or software. This is reflected in the simulator by each machine icon having different colors, and reflected in the EET (detailed below) in that tasks have different expected execution times for different machines.

    A homogeneous system means that all machines in the system are identical. This is represented in the simulator by each machine icon being the same color. In the EET, any given task will have the same expected execution times across all machines.

    A heterogeneous system may have homogeneous machines within it, e.g. multiple machines are identical to each other but some of the machines differ.

  2. Expected Execution Time (EET) Matrix

    The EET Matrix shows what task types are present in the current workload, and their respective expected execution times for each machine present. You may view and change the EET matrix by clicking on the Workload icon. To load a new EET file, click “Load,” and choose an appropriate CSV file. Be sure to also press “Submit” at the bottom. You will receive an error if the workload and EET formats are not compatible, e.g. there exists a task type in the workload not present in the EET. You may also edit individual EET values by clicking “Edit,” and then double-clicking the value you wish to change. Be sure to submit any changes.
